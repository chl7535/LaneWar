# 프로젝트: 랜덤 디펜스 로그라이크 (나랜디 + 발라트로 + 로그라이크)

## 아키텍처 원칙
- 게임 로직(데이터/규칙)과 뷰(렌더링/애니메이션)를 철저히 분리한다.
- 로직은 MonoBehaviour에 의존하지 않는 순수 C# 클래스로 최대한 분리해, 향후 멀티플레이 서버 로직 분리와 유닛 테스트가 가능하게 한다.
- 유닛/적/특성의 스탯은 ScriptableObject 데이터로 정의하고, 코드에 하드코딩하지 않는다.
- 매직 넘버 금지. 모든 수치는 데이터 또는 명명된 상수로.

## 코드 스타일
- 클래스/메서드는 단일 책임을 지킨다.
- public 필드 대신 [SerializeField] private 사용.
- 이벤트 기반 통신을 우선한다(직접 참조 최소화).
- 각 클래스 상단에 한 줄 한국어 주석으로 역할을 설명한다.

## 게임 규칙 (고정 스펙)
- 라운드: 65초 = 스폰 40초 + 정비 20초 + 카운트다운 5초
- 스폰: 초당 1마리, 라운드당 40마리
- 맵: 사각형, 적은 바깥 라인을 반시계 방향으로 순환
- 유닛: 맵 안쪽 배치, 사거리 내 적 자동공격, 마우스로 선택/이동
- 4라운드마다 보스전
- 로그라이크: 사망 시 포인트 획득 → 조커/조합식/유닛 영구 해금

## 하지 말 것
- 저작권 있는 IP(나루토 등)의 이름/캐릭터/에셋 사용 금지. 모든 콘텐츠는 오리지널.

## 프로젝트 구조
- Unity 프로젝트 루트: `LaneWar/`
- 커스텀 코드/에셋: `LaneWar/Assets/_Project/`
  - `Scripts/Core` — MonoBehaviour에 의존하지 않는 순수 C# 로직
  - `Scripts/Units` — 아군 유닛 관련 스크립트
  - `Scripts/Enemies` — 적 관련 스크립트
  - `Scripts/Systems` — 라운드/스폰/이코노미 등 게임 시스템
  - `Scripts/UI` — UI 스크립트
  - `Scripts/Data` — ScriptableObject 정의 클래스
  - `Prefabs`, `ScriptableObjects`, `Scenes`, `Art`, `Audio` — 에셋 데이터
